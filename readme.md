[안내] 도면목록표 ↔ 개별 도면 자동 검토 프로그램 (AutoDWG Cross-Checker)
========================================================================

[버전별 주요 업데이트 히스토리 (Technical Roadmap)]

▶ V1.0 : CAD Entity Text Parser (초기 모델)
 - 기술: 도면 내 전체 TEXT/MTEXT 엔티티를 전수 조사하여 단순 리스트업.
 - 한계: 불필요한 도면 요소가 섞여 데이터 신뢰도가 낮음.

▶ V1.5 : Attribute Reference Mapping
 - 기술: 일반 텍스트 외에 '블록 속성(Attribute)' 데이터를 매핑하는 기능 추가.
 - 해결: 수동 입력된 텍스트와 블록 속성을 구분하여 데이터 수집 범위 확장.

▶ V2.0 : LISP-based User ROI Selection
 - 기술: LISP 연동을 통한 사용자 지정 구역(Region of Interest) 인식 시스템 도입.
 - 해결: '어디를 읽어야 하는가'에 대한 물리적 가이드를 통해 분석 정확도 획기적 향상.

▶ V2.5 : JSON Config Persistence Engine
 - 기술: AppData 내에 도곽 블록별 구역 좌표 정보를 JSON 형태로 로컬 저장 및 관리.
 - 해결: 도면을 열 때마다 구역을 재지정해야 하는 번거로움을 없애고 프로젝트 일관성 확보.

▶ V3.0 : Coordinate Matrix Transformation
 - 기술: 삼각함수를 이용한 도곽 블록의 Rotation 및 Scale 역산 알고리즘 탑재.
 - 해결: 도면이 회전되어 있거나 스케일이 달라도 실제 좌표계로 변환하여 구역 스캔 성공.

▶ V3.5 : Vertical/Portrait Layout Detection
 - 기술: 세로형 도곽 및 변형 도곽에 대한 기하학적 보정 로직 추가.
 - 해결: 90도 회전된 세로 도면에서 텍스트가 거꾸로 읽히거나 구역을 벗어나는 문제 해결.

▶ V4.0 : Global Header Cleaner (지우개 엔진)
 - 기술: "DWG NO", "TITLE" 등 데이터가 아닌 머리글(Header)을 식별하여 제거하는 필터링.
 - 해결: 추출된 데이터에서 불필요한 영문/한글 찌꺼기를 제거하여 순수 정보만 추출.

▶ V4.5 : Regex-based Semantic Validation
 - 기술: 정규표현식(Regular Expression)을 이용한 도면번호 패턴 유효성 검증 알고리즘.
 - 해결: 단순 텍스트와 실제 도면번호 형식을 구분하여 데이터 오인식 확률 원천 차단.

▶ V5.0 : Multi-Column Zoning Support (다단 스캔)
 - 기술: 목록표의 다단(Column) 레이아웃을 독립된 데이터 세그먼트로 인식하는 알고리즘.
 - 해결: 1단, 2단으로 나뉜 도면목록표 데이터를 섞이지 않게 순차적으로 병합 분석 성공.

▶ V5.5 : Dynamic Y-Axis Row Splitting
 - 기술: 행 간격(Y-Tolerance) 자동 계산을 통한 텍스트 행 병합 및 분리 정밀도 향상.
 - 해결: 폰트 크기나 정렬 기준점(Justification) 차이로 인해 텍스트 줄바꿈이 일어나는 현상 보정.

▶ V6.0 : X-Ray XREF Overlay Technology
 - 기술: 외부참조(XREF) 원본 파일을 사전 스캔하여 현재 도면에 '가상 레이어'로 합성하는 기술.
 - 해결: 파이썬이 읽을 수 없던 외부참조 내 하이픈(-)과 축척 범례(A1/A3)를 100% 인식.

▶ V6.14 : Ultimate Refinement (최종 진화형)
 - 기술: 하이픈 자석(Hyphen Magnet) 엔진 및 중복 하이픈 압축, 특정 키워드(SUBJECT) 정밀 타겟팅 삭제.
 - 해결: 실무에서 발생하는 하이픈 누락 실수 감지 및 도면명 찌꺼기 완벽 제거를 통한 최종 QA 완성.

1. 개요
  본 프로그램은 도서 납품 전, '도면목록표'와 수백 장의 '개별 캐드 도면' 타이틀 정보를 
  자동으로 교차 검토하여 데이터 누락, 오타, 하이픈(-) 누락 등의 불일치를 
  찾아내는 업무 자동화 및 QA(품질보증) 솔루션입니다.

2. 필수 사전 설치 사항 (중요!)
  이 프로그램이 백그라운드에서 캐드 파일을 고속으로 읽어오려면 'ODA File Converter'라는 분석 엔진이 반드시 필요합니다.

  - 다운로드 링크: https://www.opendesign.com/guestfiles/oda_file_converter
  - 설치 주의: 설치 시 경로를 변경하지 마세요. (기본값: C:\Program Files\ODA)

3. 사용 방법
  [1단계] 오토캐드에서 도곽 구역 설정하기 (프로젝트당 최초 1회만 실행)
  ① 캐드에서 `SET_ROI.lsp`를 로드하고 `SET_ROI` 명령어를 실행합니다.
  ② 도곽 블록 선택 후 개별도면의 [도면번호], [도면명], [축척] 구역을 지정합니다.
  ③ [도면목록표]의 경우, 데이터가 있는 '단(Column)'을 순서대로 클릭하여 지정합니다.

  [2단계] 검토 프로그램 실행하기 (도면 수정 시 수시로 실행)
  ① `app.py`를 실행합니다.
  ② 1단계에서 클릭했던 [도곽 블록 이름]을 정확히 입력합니다.
  ③ 도곽 [외부참조(XREF) 원본 DWG 파일]의 경로를 입력합니다. (V6.0+ 핵심: 원본 스캔용)
  ④ [도면목록표 DWG 파일]과 [개별 도면 폴더] 경로를 입력합니다.
  ⑤ 완료 후 폴더에 생성된 `도면검토리포트_최종.xlsx`를 확인합니다.

4. 검토 시 주의 사항
  - 분석 결과 엑셀에서 도면번호가 `-001` 등으로 시작하거나 빨간색으로 강조된 항목은 작업자의 입력 실수(하이픈 누락, 오타 등)일 가능성이 높으므로 원본 도면을 반드시 확인하시기 바랍니다.
  - 리포트 생성 중에는 해당 엑셀 파일을 닫아두어야 덮어쓰기 오류가 발생하지 않습니다.
